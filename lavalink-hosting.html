What is Lavalink?

Lavalink is a standalone audio sending node based on Lavaplayer and JDA-Audio. It offloads heavy music processing, downloading, and encoding from your main bot process onto a separate dedicated server. If you run a music bot for Discord, you require a Lavalink node.


By using Vexanode's managed Lavalink hosting, you don't need to worry about Java version compatibility, SSL certificate setups, or high CPU usage. We handle the infrastructure, provide you with the WebSocket connection details, and guarantee 100% uptime so your users never experience audio buffering or dropouts.

Lavalink FAQ

What is the difference between Lavalink and a normal bot host? +
Lavalink is purely for audio processing. Your bot logic (commands, events) runs on a normal Bot Hosting plan, and it sends WebSocket commands to the Lavalink node to play music.
Does Vexanode support Lavalink v4? +
Yes, all our managed nodes run the latest stable release of Lavalink v4 on optimized Java 17+ environments.
Are Spotify and Apple Music supported? +
Yes! We pre-install the LavaSRC plugin on all paid nodes, which enables native searching and playback from Spotify, Apple Music, and Deezer.
What happens if my node goes offline? +
Our infrastructure includes automatic crash detection. If a node fails, it restarts within 3 seconds. For network issues, our 100% Uptime SLA guarantees compensation for any downtime.
Can I access the application.yml config file? +
Yes. Through your control panel, you have full access to the `application.yml` file to configure passwords, ratelimits, and enable/disable specific audio sources or plugins.
Do you offer free public nodes? +
Yes, we maintain free public Lavalink endpoints for developers to test their bots. However, these are shared and not recommended for large production bots due to ratelimits.
